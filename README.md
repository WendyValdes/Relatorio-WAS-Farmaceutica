☁️ Desafio DIO - Implementação de Serviços AWS
📌 Sobre o Projeto

Este repositório contém o desafio proposto pela Digital Innovation One (DIO) para o curso de Computação em Nuvem.

O objetivo foi elaborar um relatório técnico propondo a implementação de serviços da AWS em uma empresa fictícia, com foco principal na redução de custos e modernização da infraestrutura.

A empresa escolhida foi a Abstergo Industries, atuando como um hub de distribuição farmacêutica.

🎯 Objetivo

Propor a adoção de 3 serviços AWS que proporcionem:

💰 Redução de custos imediatos
⚡ Melhoria de performance
📈 Escalabilidade
🔐 Maior segurança e disponibilidade
🏗️ Arquitetura Proposta

A arquitetura foi projetada com foco em alta disponibilidade e eficiência operacional, incluindo:

Amazon CloudFront (CDN)
Application Load Balancer
Amazon EC2 (computação)
Amazon RDS (banco de dados)
Amazon S3 (armazenamento)
Backup e Lifecycle Policies

📊 O diagrama da arquitetura está disponível no relatório.

🚀 Serviços Utilizados
📦 Amazon S3
Armazenamento de documentos e relatórios
Redução de custos com servidores físicos
Alta durabilidade e escalabilidade
🖥️ Amazon EC2
Execução das aplicações da empresa
Escalabilidade sob demanda
Eliminação de infraestrutura local
🗄️ Amazon RDS
Banco de dados gerenciado
Backup automático
Alta disponibilidade
